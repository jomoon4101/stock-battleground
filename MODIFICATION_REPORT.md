# 주식 배틀그라운드 대규모 수정 완료 보고서

작성일: 2026-06-24

## 반영 결과

1. 게임 시작 전 한국어와 영어를 선택할 수 있습니다. 영어 모드는 인터페이스, 종목·시장·AI 이름, 알림과 주요 진행 문구를 영어로 표시합니다.
2. `혼자서 테스트하기`를 독립된 큰 버튼으로 만들고 `테스트`를 굵게 표시했습니다.
3. 게임 속도 선택 순서를 공식 규칙, 빠른 게임, 터보 테스트 순으로 변경했습니다.
4. 10종 주식 밈 프로필을 추가했습니다. 사용자 사진은 브라우저에서 96×96 JPEG로 축소해 사용할 수 있고, AI도 10종 중 무작위 프로필을 받습니다. 내 자산과 생존 순위에 프로필이 표시되며, 프로필 상세에서 순자산·전체 보유 종목·턴별 순위·자산 이력 그래프를 확인할 수 있습니다.
5. 1위가 바뀌거나 과거 1위가 다시 올라오면 교체·탈환 알림이 표시되고 알림함에 기록됩니다.
6. 1:1 쪽지함, 안 읽은 개수 배지, 순위 목록의 바로 보내기, 플레이어 검색, AI 답장을 추가했습니다. 쪽지는 방 메모리에만 존재하며 게임 종료 시 전부 삭제됩니다.
7. 메인 화면에 완료 게임의 순자산 상위 10명 명예의 전당을 추가하고, 이전 순위 대비 상승·하락 화살표와 수치를 표시합니다.
8. 메인 화면 하단에 `일개미(개발자)에게 일시키기` 익명 게시판을 추가했습니다.
9. 11턴 시작부터 매 턴 최하위 1명이 탈락합니다. 탈락 화면에는 순위, 매수·매도 종목과 금융 활동 기록, 거래·아이템·대출·채권 횟수, 자산 그래프, 게임 언어에 맞는 주식 명언과 관전 버튼이 표시됩니다.
10. 메인 `게임 시작`은 5초 자동 매칭을 시작합니다. 같은 언어·게임 속도 사용자를 같은 방에 넣고, 5초 뒤 100명에 못 미치는 인원을 AI로 채워 자동 시작합니다. 기존 방 코드 방식은 `친구 방 만들기`로 유지했습니다.

## 구현 대체와 이유

- 밈 프로필은 개별 파일 10개 대신 4×3 PNG 스프라이트 한 장을 사용했습니다. HTTP 요청 수와 중복 자산을 줄이고 모든 화면에서 같은 좌표 체계를 쓰기 위한 선택입니다. 실제 이미지는 10개이며 나머지 두 칸은 비워 두었습니다.
- AI 쪽지 답장은 외부 생성형 AI API 대신 언어별 게임 문구 템플릿으로 구현했습니다. 별도 API 키, 사용료, 응답 지연, 사용자 쪽지의 외부 전송 없이 즉시 테스트하기 위한 대체입니다.
- 명예의 전당과 게시판은 단일 Node 서버에서 JSON 파일로 영속화했습니다. 현재 프로젝트에 데이터베이스와 인증이 없기 때문입니다. 읽기 전용 호스팅에서는 메모리로 대체되며, 다중 서버 운영 시 Redis/PostgreSQL로 교체해야 합니다.
- Vercel은 정적 프런트만 담당합니다. 장시간 SSE 연결과 메모리 방 상태가 필요한 게임 서버는 Docker 기반 상태형 호스팅에 배포해야 합니다.

## 현재 운영 한계

- 진행 중인 방과 쪽지는 서버 재시작 시 사라집니다. 쪽지는 게임 종료 때도 의도적으로 삭제됩니다.
- 명예의 전당과 익명 게시판에는 아직 계정 인증, 신고, 운영자 삭제 기능이 없습니다.
- 여러 게임 서버 인스턴스를 함께 운영하려면 방·쪽지·전적을 공유 데이터 저장소로 이전해야 합니다.
- 로컬 인앱 브라우저 자동 제어는 실행 환경의 로컬 주소 접근 정책에 막혀 수행하지 못했습니다. 대신 API 통합 테스트, DOM 계약 테스트, 구문 검사, 빌드 산출물 HTTP 검증으로 대체했습니다.
