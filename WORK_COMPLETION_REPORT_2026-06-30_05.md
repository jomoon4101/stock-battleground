## 2. 게임 규칙 구조

- 참가 인원을 3~6명에서 선택할 수 있다. 범위를 벗어난 요청은 서버와 엔진에서 3~6으로 제한한다.
- 게임 모드:
  - 빠른 게임: 10라운드
  - 기본 게임: 20라운드
  - 장기 게임: 30라운드
- 투자 구역 난이도:
  - 쉬움: 5종목
  - 보통: 8종목
  - 어려움: 10종목
- 방 생성과 자동 매칭은 언어, 모드, 참가 인원, 난이도가 같은 사용자끼리 묶는다.
- 모든 생존 실제 참가자가 `턴 종료`를 누르면 다음 라운드로 즉시 진행한다.
- 턴 종료를 누르지 않아도 기존 서버 제한시간이 끝나면 자동 진행한다.

## 3. 게임 화면 가독성

### 생존 상태바

- 항상 보이는 네이비 상태바에 현재 라운드/최대 라운드, 생존 순위, 생존 자산, 현금, 위험도, 남은 시간을 배치했다.
- 위험도는 현재 순위와 참가 인원을 기준으로 안전·주의·위험으로 표시한다.
- 턴 종료 상태에서는 버튼을 대기 상태로 바꾸고 거래·금융·아이템 행동을 잠근다.

### 현재 선택 종목과 거래

- 현재 선택 종목을 별도 라벨과 큰 종목명으로 표시한다.
- 섹터, 현재가, 등락률, 보유 수량, 평균 매수가, 평가 손익을 한 카드에서 확인할 수 있다.
- 엔진에 종목별 `averagePrices` 원가 기록을 추가해 즉시 매수, 예약 매수, 전량 매도, 포트폴리오 셔플과 연동했다.
- 매수·매도 선택과 실행 버튼에 현재 종목명을 포함한다.
- 미보유 종목은 `매도 불가 · 보유 수량 없음`으로 표시하고 비활성화한다.
- 매도는 기존과 동일하게 현재 `selectedStock`만 사용하므로 다른 보유 종목을 잘못 매도하지 않는다.

### 내 보유 종목

- 모달에 들어가지 않아도 항상 찾을 수 있는 `내 보유 종목` 패널을 중앙 작업 영역에 추가했다.
- 종목명, 섹터, 수량, 평균 매수가, 현재가, 수익률, 평가금액, 평가 손익을 표시한다.
- 카드 클릭 시 해당 종목 거래 화면으로 이동한다.
- 미보유 상태 안내 문구를 추가했다.
- 왼쪽 투자 구역 종목 카드에도 `보유중` 뱃지를 표시한다.

### 투자 구역과 정보 센터

- 종목 목록을 간격이 있는 카드형 투자 구역으로 변경했다.
- 종목명, 섹터, 현재가, 등락률, 보유 여부, 안정·관심·위험 구역을 표시한다.
- 시장 정보, 분석 리포트, 미확인 첩보를 서로 다른 카드로 분리했다.
- 정보 카드마다 관련 섹터·종목, 신뢰도, 예상 영향, 짧은 설명, NEW 표시를 제공한다.
- 기존 찌라시는 실제 개인 메시지를 사용한다.

## 4. 대체 구현과 판단

- 외부 뉴스·증권사 API를 새로 연결하지 않았다. 현재 게임이 합성 시세를 사용하므로 실데이터를 섞으면 게임 시세와 정보가 불일치한다. 대신 시장 정보와 분석 리포트는 현재 선택 종목의 서버 동기화 시세·섹터·거래량으로 생성하고, 미확인 첩보는 기존 찌라시 메시지를 재사용했다.
- 참고 이미지는 저작물 그대로 삽입하지 않고 색상, 정보 계층, 차트·상승·황소 모티프만 코드 기반으로 재해석했다.
- 새 라이브러리는 추가하지 않았다.

## 7. 모바일 검증 범위

- 820px 이하에서 메인 단일열, 520px 이하에서 폼·프로필·모드 최소 폭 0과 단일열 설정을 적용했다.
- 게임 상태바는 3열 compact 구조, 보유 종목은 축약 카드, 정보 센터는 단일열로 전환한다.
- 헤드리스 Edge의 작은 `window-size` 캡처는 브라우저 자체 최소 레이아웃 폭 때문에 오른쪽이 잘리는 특성이 있었다. 이를 고려해 CSS에 실제 390px 기준 오버플로 방어 규칙을 추가했다.
- 마지막 390px 정밀 기기 에뮬레이션 재실행은 브라우저 실행 승인 한도에 걸려 추가 캡처하지 못했다. 정적 UI 계약과 반응형 CSS 검사는 자동 테스트에 포함했다.
