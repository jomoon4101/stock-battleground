# 주식서바이벌 게임 실행 오류 수정 작업완료 보고서

작성일: 2026-07-01

## 1. 확인된 원인

- 게임 서버와 `server.mjs` 자체는 정상적으로 실행됐습니다.
- 기존 `게임실행.cmd`가 호출하는 PowerShell 도우미는 서버를 숨김 백그라운드 프로세스로 실행했습니다.
- Codex 및 일부 Windows 실행 환경에서 `Path`와 `PATH` 환경변수가 함께 존재하면 PowerShell `Start-Process`가 환경변수 사전을 만들면서 충돌할 수 있었습니다.
- 서버 프로세스가 실행 도우미와 분리된 뒤 실행 창 종료와 함께 사라질 수 있어, 브라우저에는 서버 연결 실패 화면만 남을 수 있었습니다.
- 기존 PowerShell 파일은 UTF-8 BOM이 없어 Windows PowerShell 5에서 한글 안내가 깨질 가능성도 있었습니다.

## 2. 수정 내용

- `start-game.ps1`을 새 실행 흐름으로 교체했습니다.
  - PATH뿐 아니라 일반 설치 경로와 Codex 번들 경로에서도 Node.js를 찾습니다.
  - 서버 실행 후 최대 10초 동안 `/api/health`를 검사합니다.
  - 서버가 준비된 경우에만 게임 주소를 엽니다.
  - 실행 안내 창이 서버 프로세스의 수명을 유지합니다.
  - 게임 중 실행 안내 창을 닫으면 서버도 함께 종료되는 명확한 구조로 변경했습니다.
  - 실패 시 Node 미설치, 4173 포트 충돌, 서버 종료 코드를 표시합니다.
  - Windows PowerShell 한글 호환을 위해 UTF-8 BOM을 적용했습니다.
- `게임실행.cmd`를 개선했습니다.
  - 프로젝트 폴더로 이동한 뒤 실행합니다.
  - 실패 시 창을 유지해 오류를 확인할 수 있습니다.
  - 자동 진단용 `STOCK_SURVIVAL_NO_BROWSER=1` 모드를 지원합니다.
- `server.mjs`에 WebP MIME 타입을 추가해 섹터 CEO 이미지가 `image/webp`로 제공되게 했습니다.
- README에 게임 중 실행 안내 창을 유지해야 한다는 설명을 추가했습니다.
- 실행 도우미가 다시 깨지지 않도록 배포 테스트를 추가했습니다.

## 4. 실행 방법

1. 프로젝트 폴더의 `게임실행.cmd`를 더블클릭합니다.
2. `주식서바이벌 서버가 실행 중입니다.` 문구를 확인합니다.
3. 자동으로 열린 브라우저에서 게임을 실행합니다.
4. 게임 중에는 실행 안내 창을 닫지 않습니다.
5. 게임을 끝낼 때 실행 안내 창을 닫습니다.

`index.html`을 직접 더블클릭하면 서버 API가 없기 때문에 정상 실행되지 않습니다.
