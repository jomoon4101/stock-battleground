# 섹터 카드·뉴스 티커·종목 상세 거래창 개편 작업완료 보고서

작성일: 2026-07-01

## 1. 카드 크기와 스크롤 조작

- 모바일 카드 폭을 최대 340px, 높이를 460px로 확대했습니다.
- 데스크톱 카드 폭을 370px, 높이를 500px로 확대했습니다.
- 모바일 터치 스와이프를 지원합니다.
- PC에서 카드 영역을 마우스로 잡아 좌우로 끌 수 있습니다.
- 드래그한 뒤 마우스를 놓을 때 종목 클릭으로 잘못 처리되지 않도록 클릭 억제 로직을 추가했습니다.
- 카드 영역 위에서 마우스 세로 휠을 사용해도 가로로 이동합니다.
- 기존 좌우 화살표 버튼과 스크롤 스냅을 유지했습니다.
- 카드 선택 및 화면 갱신 후에도 가로 스크롤 위치가 유지됩니다.

## 2. 수익성·안정성·변동성 등급 표현

- `높음`: 붉은색 굵은 테두리와 붉은 글자
- `중간`: 초록색 굵은 테두리와 초록 글자
- `낮음`: 파란색 굵은 테두리와 파란 글자
- 수익성뿐 아니라 카드에 표시되는 안정성·변동성에도 같은 등급 색상 규칙을 적용했습니다.
- 4px 굵은 테두리와 외곽 강조를 사용해 작은 화면에서도 등급을 구분할 수 있습니다.

## 3. 섹터별 어두운 카드 배경

- 11개 섹터 카드에 해당 섹터 CEO 일러스트를 배경 이미지로 연결했습니다.
- 배경은 카드 전체 크기를 채우는 `cover` 방식으로 표시합니다.
- 어두운 그라데이션, 채도 감소, 밝기 감소를 적용해 전경 정보의 가독성을 유지했습니다.
- 카드 글자, 가격, 보유정보, 미니 차트 표면을 어두운 배경에 맞게 조정했습니다.
- 기존 중앙 CEO 3상태 이미지는 그대로 유지됩니다.

## 4. 생존 정보센터 뉴스 티커 통합

- 별도 큰 카드 영역이던 생존 정보센터를 주식 섹터 패널 내부로 이동했습니다.
- 시장 정보, 분석 공시, 찌라시를 한 줄 뉴스 티커 형태로 표시합니다.
- 내용이 길면 가로로 탐색할 수 있으며 각 정보 종류는 색상 라벨로 구분됩니다.
- 기존 쪽지함 열기 기능은 뉴스 티커 헤더에 유지했습니다.
- 선택한 섹터가 바뀌면 시장 정보와 공시 내용도 즉시 갱신됩니다.

## 5. 선택 종목 상세 거래창

- `NOW VIEWING 현재 선택 종목` 영역을 제거했습니다.
- 메인 게임 화면의 고정 차트·주문 패널을 제거하고 종목 상세 모달 안으로 이동했습니다.
- 섹터 카드를 클릭하면 캐릭터 정보창 형태의 종목 상세 모달이 열립니다.
- 상세창 상단에 아래 정보를 표시합니다.
  - 현재 등락에 맞는 CEO 기본·환호·낙담 이미지
  - 섹터명과 회사명
  - 섹터 설명
  - `바로 매수`, `바로 매도` 버튼
- 상세창 내부에 기존 선차트·봉차트·거래량·보유정보를 그대로 유지했습니다.
- 기존 매수·매도, 예약주문, 대출·채권, 아이템 탭도 상세창 내부에서 그대로 사용할 수 있습니다.
- 보유 종목, 순위 상세, 거래 로그에서 종목을 선택해도 동일한 상세 거래창으로 이동합니다.
- 하단 메뉴의 차트·거래 버튼도 상세창을 엽니다.

## 6. 주요 수정 파일

- `index.html`: NOW VIEWING 제거, 종목 상세 모달과 캐릭터 헤더 추가
- `app.js`: 등급 클래스, 드래그 스크롤, 뉴스 티커, 상세 모달, 기존 패널 이동 및 거래 연결
- `mobile-first.css`: 대형 카드, 섹터 일러스트 배경, 등급 색상, 뉴스 티커, 상세 거래창 반응형 디자인
- `i18n.js`: 종목 상세·즉시 매수·매도 영어 문구
- `tests/ui-contract.test.js`: 새 UI와 조작 계약 검사

## 8. 적용 시 주의사항

- 이미 열려 있는 브라우저는 강력 새로고침해야 새 CSS와 자바스크립트를 확실히 불러옵니다.
- 서버 로직은 이번 작업에서 변경하지 않았지만, 이전 작업의 제한시간 서버를 아직 재시작하지 않았다면 기존 실행 창을 닫고 `게임실행.cmd`를 다시 실행해야 합니다.
- 인앱 브라우저 localhost 자동 조작 차단으로 실제 탭 시각 검증은 실행하지 않았으며, 정적 UI 계약·통합 테스트·배포 빌드로 검증했습니다.
