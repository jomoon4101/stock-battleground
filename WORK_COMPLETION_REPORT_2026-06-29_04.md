- 작업일: 2026-06-29
- 작업 범위: 진행 중 게임 목록, 찌라시 상위권 면역, 11라운드 보유세, 메인 타이틀·아이콘 개편
- 최종 상태: 요청 기능 반영 완료, 자동 테스트·빌드·브라우저 검증 통과

## 1. 반영 내용

### 서바이벌 진행중 목록

- 메인 화면에 `🔥 서바이벌 진행중` 섹션과 수동 새로고침 버튼을 추가했다.
- `GET /api/rooms/active`가 실행 중이며 종료되지 않았고, 정원이 차지 않았으며, 인계 가능한 생존 AI 좌석이 있는 방만 반환한다.
- 방 제목, 코드, 현재 라운드, 실제 사용자 참가자 수, 정원, 진행 상태를 표시한다.
- 목록의 `바로 참여`는 기존 `joinOnlineRoom`과 기존 방 참가 API를 그대로 사용하며 방 코드만 내부 인자로 전달한다.
- 기존 6자리 참여코드 입력 방식은 유지했다.
- 5초 주기 갱신과 수동 갱신을 적용했다. 공개 목록 전용 소켓이나 새 라이브러리를 추가하지 않고 기존 HTTP API를 활용하기 위한 선택이다.
- 대기 중 방, 종료된 방, 정원 초과·AI 좌석 부족 방은 제외한다.

### 상위 5위 찌라시 면역

- `RUMOR_IMMUNE_TOP_RANK = 5` 상수를 추가했다.
- 게임 생성 및 매 라운드 정산 후 실제 순위로 `rumorImmune` 상태를 갱신한다.
- 서버와 혼자서 테스트하기 양쪽의 찌라시 발송 경로에서 `rank <= 5` 플레이어를 제외한다.
- 5위 경계를 실제로 넘을 때만 전환 이벤트를 생성하므로 같은 상태의 반복 알림이 발생하지 않는다.
- 온라인 알림은 해당 플레이어에게만 노출되며 기존 게임 알림 UI를 사용한다.

### 11라운드 보유세

- 다음 상수를 엔진 상단에 추가했다.
  - `HOLDING_TAX_START_ROUND = 11`
  - `HOLDING_TAX_THRESHOLD = 2_000_000`
  - `HOLDING_TAX_RATE = 0.02`
- 11라운드부터 현재 현금이 2,000,000원 이상인 생존 플레이어에게 `Math.floor(cash * 0.02)`를 차감한다.
- 플레이어별 `lastHoldingTaxAppliedRound`로 같은 라운드 중복 차감을 차단한다.
- `holdingTaxEligible` 상태로 신규 대상·대상 제외 전환을 추적한다.
- 대상 진입, 실제 세금 차감, 대상 제외 알림을 기존 알림 UI에 연결했다.
- 서버 알림은 플레이어별로 필터링하며, 다수 플레이어의 개인 알림이 서로를 밀어내지 않도록 내부 보관량을 확장했다.

### 메인 타이틀·아이콘

- 제목 마크업의 강제 줄바꿈을 제거하고 `주식서바이벌`을 한 줄로 구성했다.
- `white-space: nowrap`과 반응형 `clamp()` 글자 크기를 적용했다.
- 원화 문자 아이콘을 상승 라인 차트 SVG 아이콘으로 교체했다.
- 언어 변경 함수가 SVG 내용을 통화 문자로 덮어쓰지 않도록 수정했다.
- 390×844 모바일 화면에서 가로 스크롤 없이 제목과 진행 중 목록이 표시되는 것을 확인했다.

## 2. 변경 파일

## 4. 대체 구현과 미반영 항목

- 실시간 방 목록은 별도 공개 WebSocket을 추가하는 대신 5초 HTTP 폴링으로 구현했다. 새 의존성 없이 기존 서버 구조를 재사용하면서 목록 최신성을 유지하기 위한 방식이다.
- 요청 기능 중 미반영 항목은 없다.

## 5. 운영 시 주의사항

- 방 정보는 현재 Node 서버 프로세스 메모리에 있으므로 서버 재시작 시 사라진다.
- 여러 서버 인스턴스로 수평 확장하면 각 인스턴스의 진행 중 목록이 분리된다. 운영 규모에서 통합 목록이 필요하면 Redis 또는 공유 DB가 필요하다.
