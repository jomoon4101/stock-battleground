## 사용자 의도 재반영

이번 요청에서 말한 누락 내용은 정보센터 패널 자체가 아니라 패널 내부의 `[시장]`, `[공시]`, `[찌라시]` 카테고리명과 각 본문이었습니다. 공시 종류명이 카테고리 자리를 대신하던 구조를 수정했습니다.

## 완료 항목

1. 정보센터 카테고리와 본문 복원
   - 시장 정보는 `[시장] + 섹터명 + 종목·등락 정보 + 바로 거래` 순서로 표시합니다.
   - 공시 정보는 `[공시] + 공시 종류 + 섹터명 + 공시 본문 + 상세` 순서로 표시합니다.
   - 찌라시 정보는 `[찌라시] + 찌라시 본문 + 열기` 순서로 표시합니다.
   - 영어 모드에서는 `[MARKET]`, `[DISCLOSURE]`, `[RUMOR]`로 표시합니다.
   - 공시는 이전 작업대로 서로 다른 섹터의 3건을 동시에 표시합니다.

2. 카드 전체 클릭 거래창
   - 카드 외곽 `article`에도 `거래창 열기` 버튼과 동일한 `data-open-stock-detail` 액션을 부여했습니다.
   - 카드 본문 어디를 클릭해도 `openStockDetail()`을 사용해 동일한 차트·매수·매도 모달이 열립니다.
   - 카드 드래그를 완료한 직후에는 거래창이 잘못 열리지 않도록 드래그 억제 검사를 먼저 실행합니다.
   - 키보드 Enter와 Space 진입도 유지했습니다.

3. 캐시 갱신
   - 앱과 CSS 배포 버전을 `20260701-18`로 갱신했습니다.
