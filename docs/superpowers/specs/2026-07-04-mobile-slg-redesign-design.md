# 주식서바이벌 모바일 SLG 전면 리디자인 설계

작성일: 2026-07-04

## 1. 목표

현재의 밝은 웹 대시보드형 화면을 모바일 전략 게임 앱 형태로 전면 재작성한다. 모든 핵심 기능은 유지하면서 정보 과밀, 작은 글씨, 불명확한 행동 우선순위와 긴 단일 페이지 문제를 해결한다.

완료된 화면은 390px, 412px, 430px 모바일 폭에서 자연스럽게 동작하고, PC에서도 최대 480px 앱 프레임으로 중앙 정렬된다.

## 2. 승인된 결정

- 구현 방식: 프런트 화면 계층 전면 재작성(C안)
- 유지 범위: `engine.js`, `server.mjs`, 멀티플레이 API, 게임 규칙과 저장 형식
- 화면 폭: 모든 환경에서 `max-width: 480px`
- 게임 내 이동: `홈 / 시장 / 거래 / 생존자 / 로그` 5개 실제 탭
- 전체 채팅: 탭과 분리된 약 44px 말풍선 버튼, 바텀시트로 열기
- 시각 방향: 딥 네이비·차콜 배경, 레드 CTA, 골드 자산·순위, 블루 정보, 민트 안전 상태
- 새 프런트 라이브러리 추가 없음

## 3. 화면 아키텍처

### 3.1 공통 앱 셸

- `body` 중앙에 최대 480px 앱 프레임을 둔다.
- 화면 배경은 딥 네이비 그라데이션으로 통일하고 PC의 바깥 여백에는 더 어두운 배경을 사용한다.
- 기본 본문은 14px 이상, 카드 제목은 16px 이상, 주요 수치는 24px 이상으로 한다.
- 터치 가능한 컨트롤은 최소 44px, 주요 버튼은 56~64px로 만든다.
- 안전영역을 고려해 상단과 하단에 `env(safe-area-inset-*)`를 반영한다.

### 3.2 메인 화면

위에서 아래 순서로 구성한다.

1. 앱 헤더: 로고, 한 줄 타이틀, `KO / EN`
2. 히어로: 핵심 문구와 `온라인 생존전 시작`
3. 내 트레이더: 프로필, 닉네임, 플레이어 ID, 프로필 변경
4. 게임 모드: 빠른·기본·장기 모드 카드와 선택 배지
5. 배틀 입장: 방 만들기, 방 코드 참가, 혼자 테스트하기
6. 진행 중 게임: 참가 가능한 방 카드와 빈 상태
7. 접이식 안내: 투자 자문 아님 고지, 익명 게시판, 개발자 이메일

프로필 선택, 명예의 전당과 상세 안내는 처음부터 길게 펼치지 않고 모달 또는 접이식 패널로 연다.

### 3.3 매칭과 대기실

- 매칭 화면은 전체 앱 프레임 안에서 5초 카운트다운, 참가자 수와 취소 버튼만 강조한다.
- 방 대기실은 방 코드, 참가자 목록, 방장 시작 버튼을 큰 카드 하나에 모은다.
- 모바일에서 방 코드가 잘리지 않으며 클릭 복사 상태를 즉시 표시한다.

### 3.4 게임 공통 HUD

상단 고정 HUD에 다음을 통합한다.

- 현재 라운드 / 전체 라운드
- 남은 시간
- 생존 자산
- 현재 순위
- 생존 위험도
- 보유 현금

생존 자산을 가장 크게 표시하고, 순위·위험도·현금은 뱃지로 보조한다. 체크포인트 종료 10초 카운트다운은 HUD 내부에서만 강조한다.

### 3.5 게임 탭

#### 홈

- 내 트레이더 요약과 생존 목표
- 이번 라운드 행동 상태
- 최근 핵심 알림과 찌라시 도착 상태
- 보유 종목 간단 목록
- 금융·아이템·쪽지로 이동하는 큰 메뉴 버튼

#### 시장

- 섹터·회사 검색
- 가로 스크롤 섹터 칩
- 선택 종목의 CEO 이미지, 가격, 등락률, 보유 수량
- 수익성·안정성·변동성 가로 태그
- 시장·공시·찌라시를 각 한 줄로 표시하는 생존 정보 센터
- 매수·매도·상세 버튼

#### 거래

- 선택 종목과 다른 섹터를 바꿀 수 있는 상단 선택기
- 선·봉 차트, 거래량, 급등·급락 신호
- 매수·매도 수량, 빠른 비율, 예상 금액
- 예약 주문, 대출·채권과 아이템 진입 버튼
- 내 보유 종목 목록

#### 생존자

- 검색 가능한 생존 순위
- 현재 플레이어 강조
- 프로필 클릭 시 자산·순위 그래프와 보유 종목
- 플레이어별 쪽지 버튼

#### 로그

- 최근 게임 로그 3개를 우선 표시
- 거래·라운드·세금·탈락·알림 필터
- 전체 로그 펼치기
- 쪽지함과 알림 기록 진입

### 3.6 하단 고정 영역

- 최하단에는 5개 탭 내비게이션을 둔다.
- 그 위에는 `거래장 열기 / 턴 종료` 액션바를 둔다.
- 액션바는 게임 중에만 표시하고 모달이 열리면 숨긴다.
- 전체 채팅 말풍선은 오른쪽 하단에 44px 크기로 배치하고 액션바와 겹치지 않게 한다.
- 새 채팅은 작은 숫자 배지로 표시한다.

### 3.7 모달과 바텀시트

모바일에서는 아래 기능을 풀스크린 또는 90dvh 이하 바텀시트로 통일한다.

- 거래 상세
- 프로필 선택
- 쪽지함
- 금융·대출·채권
- 아이템
- 플레이어 상세
- 순위 상세
- 전체 채팅
- 규칙과 익명 게시판

닫기 버튼, 배경 클릭, `Escape` 동작을 동일하게 제공하고, 열린 동안 뒤 화면 스크롤을 잠근다.

## 4. 프런트 구조

프런트는 바닐라 ES Module을 유지하되 역할을 나눈다.

- `app.js`: 게임 상태 연결, 서버 API/SSE, 화면 전환 오케스트레이션
- `ui-shell.js`: 앱 셸, 메인·로비·게임 탭 마크업 생성
- `ui-renderers.js`: HUD, 시장, 거래, 생존자, 로그의 상태 렌더링
- `ui-actions.js`: 탭, 모달, 바텀시트와 폼 이벤트 위임
- `styles.css`: 토큰과 공통 모바일 SLG 컴포넌트
- `mobile-first.css`: 390·412·430·480px 미세 조정과 안전영역

순환 import를 피하기 위해 UI 모듈은 게임 엔진을 직접 호출하지 않는다. `app.js`가 상태와 명령 콜백을 전달한다.

## 5. 상태와 데이터 흐름

1. 로컬 게임 또는 서버 SSE가 기존 게임 상태를 갱신한다.
2. `app.js`가 상태를 화면용 뷰 모델로 정리한다.
3. 활성 탭의 렌더러만 다시 그린다.
4. 사용자의 거래·턴 종료·쪽지 행동은 이벤트 위임을 통해 `app.js` 명령으로 전달한다.
5. 서버 응답 후 새 상태를 렌더링하고 토스트로 성공·실패를 알린다.

프런트 전용 상태는 다음만 별도로 둔다.

- 활성 탭
- 선택 섹터와 종목
- 열린 모달/바텀시트
- 채팅 접힘 상태와 미확인 수
- 검색어와 로그 필터

게임 자산, 순위, 거래와 라운드 결과를 프런트 전용 상태에 복제하지 않는다.

## 6. 이미지와 폴백

- 기존 `assets/sector-ceo-*.webp`를 우선 사용하고 PNG를 대체 소스로 사용한다.
- 이미지 요소는 로딩 전에도 섹터별 그라데이션 배경을 가진다.
- 로드 오류 시 검정 박스를 남기지 않고 섹터 아이콘, 회사명과 그라데이션 폴백을 보여준다.
- 모든 경로는 프로젝트 루트 기준 상대 경로를 사용하고 `npm run build` 결과의 `dist/assets` 포함 여부를 검사한다.

## 7. 다국어와 통화

- 기존 `i18n.js`와 `setLanguage` 흐름을 유지한다.
- 새 UI의 모든 표시 문자열을 한국어·영어 사전에 등록한다.
- 영어 모드의 달러 환산과 `$` 표시는 기존 `1달러 = 1,500원` 규칙을 유지한다.
- 타이틀과 수치가 영어에서도 잘리지 않는지 480px 이하에서 확인한다.

## 8. 오류·빈 상태

- 서버 연결 전: 로딩 스켈레톤과 재시도 안내
- 진행 중 방 없음: 새 방 만들기 CTA가 있는 빈 상태
- 이미지 실패: 섹터 그라데이션과 아이콘 폴백
- 보유 종목 없음: 시장 탭 이동 버튼
- 거래 실패: 입력값을 유지하고 원인을 토스트로 표시
- SSE 재연결: HUD에 연결 상태를 표시하고 자동 재연결
- 탈락 플레이어: 기존 결과와 관전 기능을 새 풀스크린 결과 화면으로 표시

## 9. 접근성과 상호작용

- 아이콘 버튼에 한국어·영어 `aria-label`을 제공한다.
- 선택 탭은 `aria-current="page"`, 모달은 `role="dialog"`와 `aria-modal="true"`를 사용한다.
- 포커스 링과 키보드 접근을 유지한다.
- 상승은 빨강, 하락은 파랑을 유지하되 아이콘과 부호를 함께 사용해 색상만으로 상태를 구분하지 않는다.
- 모션 감소 설정에서는 폭죽·순위·토스트 애니메이션을 최소화한다.

## 10. 테스트 전략

### 자동 테스트

- 새 앱 셸과 5개 탭의 DOM 계약
- 탭 전환 시 한 화면만 표시되는지 검증
- 기존 방 생성·참가·매칭·재접속 API 회귀 테스트
- 매수·매도·턴 종료·예약 주문·금융·아이템 흐름 회귀 테스트
- 한국어·영어와 원·달러 표시 검증
- 이미지 폴백과 정적 자산 빌드 포함 검증
- `npm test`, `npm run build`, `node --check`

### 브라우저 검증

- 390px, 412px, 430px: 가로 넘침, 버튼 터치 크기, HUD·액션바·내비게이션 겹침 확인
- 480px 앱 프레임과 1280px PC 화면: 중앙 정렬과 바깥 배경 확인
- 메인 → 매칭/방 → 게임 → 5개 탭 → 거래 → 턴 종료 → 채팅 순서로 동작 확인
- 콘솔 오류와 실패한 이미지 요청 확인

## 11. 구현 순서

1. 새 UI 계약 테스트를 실패 상태로 추가
2. 전역 토큰과 480px 앱 셸 구축
3. 메인·매칭·대기실 재작성
4. 게임 HUD와 5개 탭 셸 구축
5. 기존 기능을 탭별 렌더러와 이벤트로 이전
6. 모달·바텀시트와 작은 채팅 말풍선 통일
7. 이미지 폴백과 다국어 완성
8. 전체 자동 테스트·빌드·브라우저 화면 검증
9. 작업완료 보고서 작성

## 12. 제외 범위

- 게임 규칙과 경제 수치 변경
- 서버 저장 구조와 멀티플레이 프로토콜 변경
- React/Vue 등 새 프레임워크 도입
- 외부 이미지·아이콘 라이브러리 추가
- 사운드와 진동의 실제 재생 기능

사운드·진동은 향후 연결할 수 있도록 UI 행동 이벤트 이름만 일관되게 유지한다.
