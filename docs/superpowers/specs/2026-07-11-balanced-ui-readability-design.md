# Balanced UI Readability Design

## Goal

모바일 320~480px를 기본 기준으로 게임의 모든 핵심 정보가 겹치거나 잘리지 않고, 한 번에 우선순위를 파악할 수 있도록 화면 밀도와 타이포그래피를 통일한다. 기존 다크 SLG 분위기와 게임 기능은 유지한다.

## Approved Direction

균형형 밀도를 사용한다.

- 일반 본문과 조작 문구: 14~16px
- 보조 설명: 최소 12px
- 핵심 금액·순위·현재가: 18~26px
- 터치 대상: 최소 44px
- 카드 높이: 콘텐츠 기반 `min-height`, 불필요한 고정 `height` 제거
- 긴 텍스트: 단어 단위 줄바꿈을 우선하고 숫자·금액은 줄바꿈 금지

## Layout

### Start Screen

- 언어 버튼은 우측 상단의 작은 독립 컨트롤로 유지하되 타이틀과 겹치지 않는다.
- 타이틀은 한 줄을 유지하면서 320~480px에서 잘리지 않는 반응형 크기를 사용한다.
- 네 가지 소개 카드는 2열 그리드로 통일하고 제목과 설명을 서로 다른 줄에 둔다.
- 입력, 프로필, 게임 모드, 방 생성/참여는 각자 명확한 그룹 간격을 가진다.

### Battle HUD

- 상단 첫 줄은 브랜드, 라운드·타이머, 방 상태·필수 조작만 표시한다.
- 생존 상태 여섯 항목은 모바일에서 3열 2행으로 고정한다. 412px 이상에서도 6열로 압축하지 않는다.
- 라벨은 한 줄, 값은 한 줄을 원칙으로 하고 금액 단위를 임의로 분리하지 않는다.

### Home

- 내 정보 카드는 프로필/이름, 현재 순위, 쪽지·알림을 첫 행에 둔다.
- 내 목표는 카드 전체 폭을 차지하는 둘째 행으로 분리한다.
- 자산과 보유 종목 카드는 동일한 헤더 높이, 좌우 여백, 제목 크기를 사용한다.
- 빈 상태는 과도한 높이를 사용하지 않고 안내와 한 개의 행동 버튼만 보여준다.

### Market Cards

- 가로 스크롤과 스냅 동작은 유지한다.
- 섹터 카드는 고정 높이 대신 최소 높이를 사용하고 내부 세로 오버플로를 허용하지 않는다.
- 섹터명, 회사명, 현재가, 등락률을 우선 정보로 두고 수익성/안정성/변동성 및 보유정보는 읽을 수 있는 최소 크기를 보장한다.
- 생존 정보 센터는 시장/공시/찌라시 각 최신 한 건의 3행 구조를 유지한다.

### Trade

- 종목명과 현재가는 헤더 양쪽에 배치하되 서로 침범하지 않는다.
- 보유정보 네 칸은 2열 2행을 기본으로 사용한다.
- 탭은 가로 스크롤 또는 균등 배치로 글자가 세로로 쪼개지지 않게 한다.
- 매수/매도, 수량, 비율, 주문금액, 실행 버튼을 세로 흐름으로 명확히 분리한다.

### Fixed Controls

- 턴 행동 바와 하단 메뉴의 높이·아이콘·라벨 정렬을 통일한다.
- 콘텐츠 하단 패딩은 두 고정 바의 실제 높이를 포함해 마지막 카드가 가려지지 않게 한다.

## CSS Strategy

- 기존 HTML 구조와 상태 ID를 유지한다.
- `mobile-first.css` 마지막에 하나의 명시적인 `Balanced readability system` 계층을 두어 이전 규칙을 안정적으로 재정의한다.
- 공통 간격과 글자 크기는 CSS 변수로 관리한다.
- 필요한 경우에만 `ui-shell.js`의 그룹 래퍼를 보완하고 게임 상태/이벤트 로직은 변경하지 않는다.

## Verification

- CSS 계약 테스트로 핵심 크기, 3열 상태바, 전체 폭 목표 카드, 자동 높이 섹터 카드와 최소 터치 크기를 고정한다.
- 전체 Node 테스트와 프로덕션 빌드를 실행한다.
- 실제 브라우저에서 360x800, 390x844, 480x900을 확인한다.
- DOM 측정으로 가로 오버플로, 섹터 카드 내부 세로 오버플로, 비정상적으로 좁은 텍스트 블록이 없는지 확인한다.
- 콘솔 오류가 없는지 확인한다.
