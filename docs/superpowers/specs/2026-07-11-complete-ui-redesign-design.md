# 주식서바이벌 전체 UI 리디자인 명세

## 1. 목표

기존 기능과 화면 ID를 유지하면서 메인, 매칭, 로비, 게임 탭, 거래 상세, 순위, 로그, 쪽지, 알림, 탈락, 승리 화면을 하나의 모바일 우선 디자인 시스템으로 통일한다. 사용자가 직접 플레이어 수를 선택하는 UI는 제거하고, 게임 모드가 인원과 라운드를 자동 결정한다.

## 2. 승인된 시각 방향

기존 대화에서 선택된 C형을 기준으로 한다. 핵심은 다음과 같다.

- 어두운 금융 게임 배경과 밝고 명확한 정보 계층
- 카드 기반 대시보드, 8px 간격 체계, 일관된 12~24px 모서리
- 중립색을 기본으로 사용하고 상승·매수는 빨강, 하락·매도는 파랑, 보상·순위는 금색으로 제한
- 장식보다 수치, 행동, 상태의 가독성을 우선
- 모바일은 하단 탭과 바텀시트, 데스크톱은 넓은 카드 그리드와 고정 내비게이션

이 방향은 Figma에서 흔히 쓰이는 재사용 컴포넌트 기반 앱 구조와 공식 모바일 디자인 지침의 카드, 탭 바, 시트, 명확한 타이포그래피 패턴을 따른다.

## 3. 플레이어 인원 규칙

- 메인 화면에서 `solo-player-count` 선택기와 관련 라벨을 완전히 제거한다.
- 빠른 게임은 3명, 기본 게임은 5명, 장기 게임은 6명을 자동 사용한다.
- 혼자 테스트, 자동 매칭, 방 만들기 모두 선택된 모드의 `GAME_MODES[mode].playerCount`를 사용한다.
- 엔진과 서버의 `playerCount` 필드는 게임 상태 표현과 검증을 위해 유지한다.
- 참여 코드 및 진행 중 게임 바로 참여 기능은 유지한다.

## 4. 디자인 토큰

CSS 토큰은 한곳에서 관리한다.

- 배경: `#07101f`, 보조 배경: `#0d1728`
- 카드: `#111d30`, 부유 카드: `#17243a`
- 본문: `#f6f8fc`, 보조 본문: `#a9b6ca`, 비활성: `#708099`
- 상승·매수: `#ff3b5c`, 하락·매도: `#2f8cff`, 성공: `#20c997`, 보상: `#f5b942`
- 간격: 4, 8, 12, 16, 20, 24, 32px
- 글자: 보조 12~13px, 본문 15~16px, 카드 제목 18~20px, 화면 제목 24~32px
- 모바일 터치 영역은 최소 44px

## 5. 화면별 구조

### 메인

- 상단 브랜드, 언어 버튼, 한 문장 설명을 짧게 배치한다.
- 닉네임과 프로필을 하나의 설정 카드로 묶는다.
- 게임 모드는 3개의 선택 카드로 표시하며 인원과 라운드는 자동 규칙 안내로만 보여준다.
- 주요 행동은 `멀티게임`, `방 만들기`, `방 참가` 순으로 명확히 구분한다.
- 혼자 테스트는 낮은 강조도의 텍스트 버튼으로 유지한다.
- 진행 중 서바이벌과 명예의 전당은 별도 카드로 분리한다.

### 매칭·로비

- 매칭 진행률, 남은 시간, 참가 현황을 한눈에 보이는 중앙 카드로 구성한다.
- 로비는 방 코드, 참가자 목록, 시작 행동 순으로 배치한다.
- 온라인 상태와 방장 여부는 짧은 배지로 표현한다.

### 게임 공통

- 상단 HUD는 현재 라운드, 남은 시간, 순위, 자산, 위험도를 두 줄 이내로 보여준다.
- 모바일 하단 탭은 홈, 시장, 거래, 순위, 로그를 항상 같은 위치에 유지한다.
- 화면마다 하나의 대표 제목과 하나의 대표 행동만 우선 강조한다.
- 모든 숫자는 줄바꿈되지 않게 하고 단위를 보조 크기로 분리한다.

### 홈

- 플레이어, 목표, 순위와 알림을 상단 요약 카드로 통합한다.
- 자산 카드에서 현금, 주식, 채권, 대출을 동일한 구조로 표현한다.
- 보유 종목과 특수 자산은 접거나 상세 화면으로 열 수 있게 한다.

### 시장

- 검색과 생존 정보 센터를 먼저, 섹터 카드 레일을 다음에 배치한다.
- 섹터 카드는 이름, 가격, 변화율, CEO, 수익성·안정성·변동성만 남겨 정보 밀도를 제한한다.
- 카드 전체를 누르면 거래 상세 시트가 열린다.

### 거래

- 종목명과 현재가, 차트, 내 포지션, 주문 입력, 주요 행동 순으로 배치한다.
- 매수와 매도는 색상뿐 아니라 텍스트와 아이콘으로도 구분한다.
- 모바일에서는 주문 버튼을 하단 가까이에 고정하고 데스크톱에서는 차트 옆 주문 카드로 배치한다.

### 순위·로그·메시지

- 순위는 순번, 프로필, 닉네임, 자산, 변화만 한 행에 표시한다.
- 로그와 정보 센터는 가장 최신 항목이 먼저 보이게 한다.
- 메시지, 알림, 프로필, 규칙, 아이템은 동일한 시트/대화상자 헤더와 닫기 패턴을 사용한다.

### 탈락·승리

- 결과, 최종 순위, 핵심 자산, 그래프, 다시 시작 행동 순으로 구성한다.
- 승리 조건과 획득 보상은 별도 배지로 명확히 보여준다.

## 6. 텍스트 원칙

- 깨진 문자열이나 의미 없는 영문 장식을 제거한다.
- 제목은 15자 이내의 명사형, 설명은 한두 문장으로 제한한다.
- 한 화면에서 동일한 뜻의 한국어와 영어를 중복 표시하지 않는다.
- 버튼은 결과가 분명한 동사형으로 쓴다: `게임 시작`, `방 만들기`, `매수 주문`, `턴 종료`.
- 한국어와 영어 모두 레이아웃이 깨지지 않도록 유연한 높이와 말줄임을 사용한다.

## 7. 컴포넌트 경계

- 기존 동작을 보호하기 위해 DOM ID와 이벤트 연결은 최대한 유지한다.
- `ui-shell.js`는 화면 구조와 문구를 담당한다.
- `mobile-first.css` 끝의 단일 리디자인 계층이 디자인 토큰과 화면별 레이아웃을 담당한다.
- `app.js`는 플레이어 수 자동 결정처럼 동작 변경이 필요한 부분만 수정한다.
- 새로운 라이브러리는 추가하지 않는다.

## 8. 오류·접근성 처리

- 내용이 길면 겹치지 않고 줄바꿈하거나 스크롤한다.
- 고정 하단 내비게이션과 안전 영역이 콘텐츠를 가리지 않게 여백을 확보한다.
- 포커스 표시, 44px 터치 영역, 충분한 명도 대비를 적용한다.
- 빈 상태, 로딩, 연결 끊김, 마감된 방은 각각 설명과 다음 행동을 제공한다.
- 모션 감소 설정에서는 장식 애니메이션을 줄인다.

## 9. 검증 기준

- 메인 화면에 플레이어 수 입력 또는 선택기가 없다.
- 모드별 자동 인원이 혼자 테스트, 매칭, 방 만들기에 동일하게 적용된다.
- 360px 모바일부터 넓은 데스크톱까지 가로 스크롤과 텍스트 겹침이 없다.
- 메인, 매칭, 로비, 홈, 시장, 거래, 순위, 로그, 주요 모달, 탈락, 승리 장면이 같은 디자인 토큰을 사용한다.
- 기존 게임 생성, 참여, 진행, 거래, 턴 종료 기능이 유지된다.
- 전체 자동 테스트와 빌드가 성공한다.
