# 다크 UI·섹터 카드·생존정보센터 재설계 명세

작성일: 2026-07-05  
승인안: C — 스타일시트 전면 정리

## 1. 목표

현재 모바일 게임 화면에서 라이트 테마 규칙이 뒤섞여 패널 헤더와 메뉴가 흰색으로 표시되는 문제를 제거한다. 동시에 알림 배지, 섹터 카드, 생존정보센터를 모바일에서 즉시 읽을 수 있는 구조로 정리한다.

이번 작업은 프런트엔드 표현 계층에 한정한다. `engine.js`, `server.mjs`, 거래·라운드·멀티플레이 데이터 의미와 기존 DOM ID는 유지한다.

## 2. 스타일 구조

### `styles.css`

- 다크 색상 토큰, 타이포그래피, 공통 패널, 버튼, 입력창, 카드, 모달의 기본 표현을 단일 기준으로 정의한다.
- 기존 라이트 테마 블록과 동일 선택자를 반복해서 덮어쓰는 규칙을 제거한다.
- 모든 `.panel`과 `.panel-header`는 딥 네이비·차콜 배경과 밝은 본문색을 사용한다.
- 검색창, 자산 패널, 보유 종목, 생존자 순위, 시장 헤더도 같은 다크 표면 체계를 사용한다.

### `mobile-first.css`

- 320~480px 모바일 배치, safe-area, 고정 HUD·액션바·하단 내비게이션, 시트와 카드의 반응형 크기만 담당한다.
- 481px 이상에서는 앱 프레임을 최대 480px로 중앙 정렬한다.
- 색상·컴포넌트 외형을 다시 정의하는 중복 규칙을 두지 않는다.

## 3. 쪽지·알림 배지

- `#mail-unread`, `#notice-unread`, `#global-chat-unread`는 최소 18×18px 원형 배지를 사용한다.
- 숫자는 최소 11px, 굵기 900으로 표시한다.
- 아이콘 우측 상단에 배치하며 두 자리 숫자도 잘리지 않도록 `min-width`와 좌우 여백을 둔다.
- 읽지 않은 항목이 0이면 기존처럼 숨긴다.

## 4. 섹터 카드

- 카드 내부의 별도 `거래창 열기` 버튼을 제거한다.
- 카드 전체의 클릭, 키보드 Enter·Space, `data-open-stock-detail` 동작은 유지한다.
- 섹터 이름은 기존 버튼이 있던 카드 왼쪽 상단에 가독성 높은 오버레이 라벨로 배치한다.
- 회사명, 현재가, 변동률, 수익성·안정성·변동성, CEO 이미지는 기존 정보를 유지한다.
- 섹터 이름과 CEO 이미지가 겹치지 않도록 상단 안전 여백과 어두운 그라데이션을 둔다.
- 한국어·영어 접근성 라벨은 카드 전체의 거래 진입 의미를 계속 전달한다.

## 5. 생존정보센터

생존정보센터는 다음 고정 순서의 3행으로 표시한다.

1. `[시장]`
2. `[공시]`
3. `[찌라시]`

각 행에는 가장 최신 정보 1개만 표시한다. 과거 정보 가로 스크롤과 여러 카드 누적은 제거한다.

### 표시 내용

- 시장: `섹터 · 상승/하락/보합 · 변동률`
- 공시: `공시 핵심 태그 · 섹터`
- 찌라시: `섹터 · 상승/하락 암시`

원문 문장 전체는 표시하지 않고 핵심 키워드만 남긴다. 행 높이는 약 40~44px로 제한하고, 찌라시 행이 다른 행보다 커지지 않게 한다. 관련 섹터가 있으면 행을 눌러 기존 종목 상세·거래 흐름으로 이동한다. 찌라시가 없으면 짧은 빈 상태 문구를 표시하고 쪽지함 버튼은 정보센터 제목 영역에 작게 유지한다.

최신 기준은 다음과 같다.

- 시장: 현재 선택 종목의 가장 최근 턴 변동
- 공시: 계산된 공시 항목 중 가장 최근 `reportTurn`
- 찌라시: 현재 플레이어가 받은 찌라시 중 가장 최근 `createdAt`, 없으면 배열의 마지막 항목

## 6. 오류 처리와 접근성

- 섹터 이미지 로드 실패 시 기존 섹터 아이콘·회사명 폴백을 유지한다.
- 카드 전체는 포커스 가능하며 키보드로 거래창을 열 수 있어야 한다.
- 상승은 붉은색, 하락은 푸른색, 보합은 중립색을 유지하되 색상만으로 의미를 전달하지 않고 텍스트 키워드를 함께 표시한다.
- 새 문구는 한국어·영어 모두 제공하고 언어를 다시 전환해도 원문이 복원되어야 한다.
- 배지에는 읽지 않은 개수를 설명하는 접근성 이름을 유지한다.

## 7. 테스트와 검증

구현은 TDD로 진행한다.

- 라이트 배경 규칙이 남아 있지 않고 최종 패널 표면이 다크 토큰을 사용하는지 검사한다.
- 배지가 최소 18px·11px인지 검사한다.
- 섹터 카드에 중복 거래 버튼이 없고 카드 전체 거래 동작과 섹터 라벨이 존재하는지 검사한다.
- 생존정보센터가 `[시장]`, `[공시]`, `[찌라시]` 3행을 정확히 한 번씩, 최신 항목 1개씩 렌더링하는지 검사한다.
- 한국어·영어 키워드와 접근성 라벨을 검사한다.
- 전체 Node 테스트, 구문 검사, 프로덕션 빌드와 `git diff --check`를 실행한다.
- 가능하면 실제 390·412·430px 모바일과 1280px 데스크톱에서 흰색 표면, 텍스트 겹침, 배지 가독성, 정보센터 높이를 확인한다.

## 8. 제외 범위

- 게임 규칙, 가격 계산, AI 투자 판단, 서버 API와 SSE 프로토콜
- 새로운 섹터·회사·CEO 이미지 생성
- 쪽지·알림 데이터 모델 변경
- 정보센터 과거 기록 보관 화면 추가
