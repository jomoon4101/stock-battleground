# 주식 서바이벌 균형형 UI 가독성 개선 작업완료 보고서

작업일: 2026-07-11

## 완료 내용

### 시작 화면

- `주식서바이벌` 타이틀이 모바일에서 잘리거나 언어 버튼과 겹치지 않도록 크기와 여백을 조정했습니다.
- 시장 분석, 투자 전략, 섹터 정보, 커뮤니티 소개 카드를 2열로 복원했습니다.
- 닉네임 입력, 프로필, 게임 모드, 방 참여 영역의 글자 크기와 터치 높이를 통일했습니다.
- 게임 모드는 모바일에서도 세 버튼을 한 행에 유지하면서 제목과 설명을 읽을 수 있게 조정했습니다.

### 상단 HUD

- 생존 상태 6개 항목을 3열 2행으로 고정했습니다.
- 라벨 12px, 값 15px를 적용하고 금액과 시간을 한 줄로 유지했습니다.
- 좁은 화면에서 상단 기능이 타이머를 밀어내지 않도록 필수 기능 중심으로 정리했습니다.

### 홈 화면

- 내 정보 카드를 `프로필·이름 / 순위·알림 / 내 목표` 구조로 재배치했습니다.
- 세로로 찌그러졌던 `MY GOAL · 내 목표`를 카드 전체 폭으로 확장했습니다.
- 자산과 보유 종목 카드의 헤더 높이, 좌우 여백, 제목 크기를 통일했습니다.
- 빈 보유 종목 영역의 과도한 높이를 줄이고 안내와 행동 버튼을 정돈했습니다.

### 시장과 섹터 카드

- 섹터 카드의 고정 높이를 제거하고 콘텐츠 기반 높이로 변경했습니다.
- 카드 내부 수익성·안정성·변동성 및 보유정보 글자를 각각 11px 이상으로 조정했습니다.
- 회사명, 현재가, 등락률과 섹터명을 우선순위에 맞게 배치했습니다.
- 가로 스크롤과 스냅 동작은 유지하면서 넓은 화면에서는 카드 폭이 함께 확장되도록 변경했습니다.
- 생존 정보 센터 헤더를 2열 그리드로 정리해 `찌라시 쪽지함` 문구가 잘리지 않도록 수정했습니다.

### 거래 화면

- 종목명과 현재가가 서로 침범하지 않는 헤더 그리드를 적용했습니다.
- 섹터, 보유 수량, 평균 매수가, 평가 손익을 실제 2열 2행으로 배치했습니다.
- 각 라벨과 값이 붙어 보이지 않도록 값을 독립된 줄로 분리했습니다.
- 거래 탭은 글자가 세로로 쪼개지지 않는 가로형 탭으로 정리했습니다.
- 매수/매도, 수량, 비율, 주문금액, 실행 버튼을 한 방향으로 읽히는 흐름으로 통일했습니다.

### 고정 조작부

- 거래/턴 종료 바와 하단 메뉴의 최소 터치 높이를 48px로 통일했습니다.
- 버튼 제목과 설명의 크기, 정렬, 하단 콘텐츠 여백을 정리했습니다.

## 원인과 해결 방식

기존 CSS에는 그리드 열 정의만 있고 `display:grid`가 빠진 영역이 있었습니다. 이 때문에 시작 화면 소개 카드가 한 줄씩 쌓이고, 거래 보유정보가 2열 설정을 무시한 채 전체 폭으로 늘어났습니다. 또한 412px 이상에서 생존 상태를 6열로 압축하는 규칙과 내 목표 카드의 그리드 위치 누락이 글자 밀림을 만들었습니다.

기존 상태 ID와 JavaScript 로직은 유지하고, `mobile-first.css` 마지막에 `Balanced readability system` 계층을 추가해 충돌하는 이전 규칙을 한곳에서 재정의했습니다.
