# 주식 서바이벌 모바일 SLG 리디자인 완료 보고서

작성일: 2026-07-05
상태: `DONE_WITH_CONCERNS` — 구현과 자동 검증은 완료했지만 필수 브라우저 화면·동작 검증은 아직 완료하지 못했습니다.

## 1. 구현된 화면과 동작

- 기존 밝은 대시보드를 딥 네이비·차콜 기반의 최대 480px 모바일 SLG 앱 셸로 교체했습니다. PC에서는 앱 프레임을 480px로 제한해 중앙에 배치합니다.
- 게임 화면을 실제 `홈 / 시장 / 거래 / 생존자 / 로그` 5개 탭으로 분리하고, 활성 탭만 렌더링·표시하도록 연결했습니다.
- 게임 중 상단 HUD, `거래 / 턴 종료` 고정 액션바와 5탭 고정 하단 내비게이션을 제공합니다.
- 전체 채팅은 하단 44px FAB와 미확인 배지로 열며, 간결한 바텀시트에서 기존 메시지 목록·입력·전송 기능을 사용합니다.
- 메인, 공개 매칭, 방 대기실, 게임 탭, 종목 상세, 프로필, 쪽지, 알림, 아이템, 결과·탈락 화면을 새 셸에 연결했습니다.
- 첫 게임 진입 시에만 4단계 온보딩을 표시합니다. 로컬 저장소가 없거나 접근 예외가 발생해도 게임 진입은 계속됩니다.
- 11개 섹터의 CEO WebP 아트를 빌드에 포함하고, 로드 오류 시 검정 빈 화면 대신 섹터 아이콘과 회사명을 유지하는 폴백을 사용합니다.
- 한국어와 영어 런타임 라벨을 양방향으로 다시 적용할 수 있도록 보존했으며, 영어 모드의 기존 달러 환산 규칙도 유지했습니다.

## 2. 보존한 기능

이번 리디자인은 프런트 화면 계층을 변경한 작업입니다. 다음 기존 동작과 데이터 의미를 유지했습니다.

- 게임 규칙, 라운드·탈락·세금·월급·보유세·대출·채권·아이템과 AI 의사결정
- 방 만들기, 코드 참가, 공개 매칭, SSE 동기화, 재접속과 AI 좌석 중도 참가
- 즉시·예약 매수/매도, 차트, 보유 종목, 금융·아이템과 서버 판정 흐름
- 전체 채팅, 1:1 쪽지, 찌라시, 알림, 익명 게시판과 프로필·순위 상세
- 한국어/영어 전환, 게임 데이터 현지화, 원/달러 표시
- 배포 구조, 환경변수, 상태형 서버 운영 범위와 프로토타입 게임 규칙 문서

`engine.js`와 `server.mjs`의 게임·서버 규칙은 이 문서 커밋에서 변경하지 않았습니다.

## 4. 자동 검증

- 구문 검사: `app.js`, `ui-shell.js`, `ui-state.js`, `sector-art.js`, `onboarding-state.js`, `i18n.js` 모두 통과
- 자동 테스트: 80개 통과, 실패 0개
- 프로덕션 빌드: `npm.cmd run build` 성공, `Build complete: dist` 확인
- 정적 검사: `git diff --check` 통과

자동 테스트는 5개 실제 탭과 활성 패널, 최대 480px 셸, 14px 본문, 48px 탭, 44px 채팅 FAB, 고정 HUD·액션바·하단 내비게이션, safe-area CSS, 390/412/430/481px 미디어 계약, 바텀시트 상태, 온보딩 저장소 예외 처리, 11개 CEO 아트와 폴백, 가역적 한국어/영어 라벨, 필수 `dist` 모듈·정적 자산 생성을 별도로 검사합니다. 이 계약 검사는 실제 브라우저의 픽셀 배치나 사용자 흐름 검증을 대신하지 않습니다.

## 5. 브라우저 검증 제한

인앱 브라우저에 열려 있는 로컬 게임 탭이 없었고 자동 탐색도 사용할 수 없었습니다. 따라서 390px, 412px, 430px 모바일 화면과 1280px PC 화면의 시각 검증 및 브라우저 게임 흐름 검증은 완료하지 않았습니다. 통과했다고 주장하지 않습니다.

정확한 수동 후속 체크리스트:

1. `npm.cmd run dev`를 실행하고 `http://127.0.0.1:4173/`을 엽니다.
2. 390px에서 제목 한 줄, 14px 이상 본문, 48px 이상 주요 컨트롤, 방 만들기·참가 동작, HUD·액션바·하단 내비게이션 비겹침, 실패 이미지 없음과 `document.documentElement.scrollWidth === document.documentElement.clientWidth`를 확인합니다.
3. 412px와 430px에서 카드가 가용 폭을 사용하고 5개 탭 정보 구조, safe-area 여백, 고정 요소가 서로 겹치지 않는지 각각 확인합니다.
4. 1280px에서 앱이 최대 480px로 중앙 정렬되고 바깥 배경이 장식 영역으로만 남는지 확인합니다.
5. 혼자 테스트하기로 `시작 → HUD → 홈 → 시장 → 거래 → 생존자 → 로그 → 종목 선택 → 거래장 열기 → 매수 → 턴 종료 → 전체 채팅 열기 → Escape로 닫기` 흐름을 실행합니다.
6. 첫 게임에서 온보딩이 한 번 표시되고 확인 후 재진입에서는 다시 표시되지 않는지 확인합니다.
7. 한국어 → 영어 → 한국어 전환 후 탭, 동적 순위·거래 라벨과 원/달러 표시가 가역적으로 복원되는지 확인합니다.
8. 개발자 도구 Network에서 11개 CEO 이미지 요청 실패가 0인지 확인하고, 한 이미지 오류를 강제로 발생시켜 섹터 아이콘·회사명 폴백을 확인합니다.
9. 전체 흐름 뒤 브라우저 콘솔 오류가 0인지 확인합니다.

## 6. 남은 우려 사항

- 위 브라우저 확인 전에는 실제 글자 잘림, 고정 요소 겹침, 가로 스크롤, 포커스·Escape 동작과 이미지 네트워크 상태를 최종 승인할 수 없습니다.
- 자동 계약은 지정된 CSS 수치와 DOM·빌드 구조의 존재를 보장하지만, 운영 브라우저·기기별 렌더링 차이는 수동 확인이 필요합니다.
