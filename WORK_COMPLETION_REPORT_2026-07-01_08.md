# 주식서바이벌 섹터 카드 개편 작업완료 보고서

작성일: 2026-07-01

## 1. 완료 범위

- 11개 섹터마다 서로 다른 오리지널 CEO 캐릭터를 제작했습니다.
- 모든 CEO는 기본, 주가 상승 시 환호, 주가 하락 시 낙담의 3가지 상태를 가집니다.
- 실제 유명인의 얼굴을 그대로 복제하지 않고, 각 산업을 대표하는 유명 경영자·과학자·창업가의 직업적 인상을 모티브로 한 가상 인물로 구성했습니다.
- 종목 카드 한 장 안에 아래 정보를 통합했습니다.
  - 섹터 아이콘과 섹터명
  - 현재가와 등락률
  - 수익성, 안정성, 변동성
  - 섹터별 CEO 상태 이미지
  - 보유수량, 평균 매수가, 평가손익
  - 가상 회사명과 섹터 설명
  - 최근 섹터 가격 흐름 미니 선차트
- 카드 클릭 시 기존과 동일하게 해당 종목 상세·매수·매도 화면으로 이동합니다.

## 2. 섹터별 캐릭터 방향

| 섹터 | 캐릭터 모티브 |
|---|---|
| IT | 검은 터틀넥과 원형 안경의 기술 창업가 |
| 금융 | 연차보고서를 든 노련한 가치투자자 |
| 헬스케어 | 연구와 진료를 겸하는 의사과학자 |
| 경기소비재 | 럭셔리·자동차 산업의 패션 감각 있는 경영자 |
| 필수소비재 | 식품·유통 기업을 이끄는 생활밀착형 경영자 |
| 산업재 | 공장과 중장비 현장을 지휘하는 엔지니어 |
| 커뮤니케이션 | 애니메이션·방송 산업의 창작자형 경영자 |
| 원자재 | 광산과 소재 산업을 이끄는 현장형 경영자 |
| 에너지 | 석유와 신재생에너지를 함께 다루는 에너지 경영자 |
| 유틸리티 | 전력망과 발명을 상징하는 전기 개척자 |
| 부동산 | 도시 개발과 건축을 이끄는 건축가형 경영자 |

## 3. 이미지 자산

- Codex 내장 이미지 생성 기능으로 제작했습니다.
- 공통 프롬프트 방향: 동일 캐릭터의 중립·환호·낙담 3분할, 섹터 배경, 게임 카드 일러스트, 텍스트·로고·워터마크 제외, 실제 인물의 정확한 얼굴 복제 제외.
- 웹에서 실제 사용하는 최종 파일: `assets/sector-ceo-*-v2.webp` 11개
- 보존용 생성 원본: `assets/sector-ceo-*-v2.png` 11개
- WebP는 모바일 로딩을 위해 1,200px 폭으로 최적화했습니다.
- 최종 WebP 전체 용량: 1,808,884바이트(약 1.73MiB)

## 4. 수정 파일

- `app.js`
  - 섹터별 CEO 자산 매핑
  - 등락에 따른 3상태 전환
  - 카드 보유정보와 미니 선차트 생성
- `mobile-first.css`
  - 모바일 우선 트레이딩 카드 레이아웃
  - 11개 섹터별 이미지와 색상 연결
  - 상승·하락 강조, 평가손익 색상, 차트 스타일
- `scripts/build.mjs`
  - 누락되어 있던 `mobile-first.css` 배포 복사 추가
  - 11개 WebP 섹터 이미지의 `dist/assets` 복사 추가
- `tests/ui-contract.test.js`
  - 11개 이미지, 카드 정보, 미니 차트 계약 검사 추가
- `tests/deployment.test.js`
  - 배포 CSS·섹터 이미지 복사 계약 검사 추가

## 5. 검증 결과

- `node --check app.js`: 통과
- 전체 자동 테스트: 30/30 통과
- `npm run build`: 통과
- `dist/mobile-first.css`: 생성 확인
- `dist/assets/sector-ceo-*-v2.webp`: 11개 생성 확인
- `dist/index.html`, `dist/404.html`: 생성 확인

자동 테스트 중 기존 멀티플레이 순위 난수에 의한 찌라시 상태 테스트가 한 번 일시적으로 실패했으나, 코드 변경 없이 재실행하여 전체 30개가 통과했습니다.

## 6. 미완료 또는 제한 사항

- 인앱 브라우저가 현재 `http://127.0.0.1:4173` 접근을 사용자가 차단한 상태로 판정하여, 모바일 390px·데스크톱 실제 탭 시각 검증은 보안 정책상 실행하지 않았습니다.
- 우회 브라우저나 별도 자동화는 사용하지 않았습니다.
- 코드·CSS·자산·배포 산출물 검증은 완료됐으며, 브라우저 차단이 해제되면 실제 화면의 최종 간격과 이미지 초점만 추가 확인하면 됩니다.

## 7. 구현 판단

- 유명인의 얼굴을 그대로 재현하면 초상권 문제와 캐릭터 일관성 문제가 생길 수 있어, 사용자가 요청한 유명인 모티브는 산업별로 알아보기 쉬운 오리지널 인물 아키타입으로 구현했습니다.
- 생성 PNG 11개를 그대로 배포하면 약 27MiB가 추가되므로, 모바일 게임의 초기 로딩을 고려해 약 1.73MiB의 WebP 11개를 실제 게임에서 사용하도록 변경했습니다.
